# Treasurer Agent — Privacy Policy

**Last Updated:** May 14, 2026

This Privacy Policy explains how Hush Line handles information related to the Hush Line Treasurer Agent software (“Software”).

---

## 1. Overview

The Treasurer Agent is designed primarily as a locally operated financial automation tool.

By default:
- financial data is processed locally on systems controlled by the User;
- Hush Line does not automatically collect bookkeeping or banking data;
- Users control integrations and data flows.

This Privacy Policy describes:
- what information may be processed;
- when Hush Line may receive information;
- User responsibilities regarding sensitive financial data.

---

## 2. Information Processed by the Software

Depending on configuration, the Software may process:

### Financial Data
- transactions;
- invoices;
- receipts;
- bookkeeping records;
- reconciliation data;
- payroll-related metadata;
- nonprofit accounting records;
- tax-related documents.

### Account and Integration Data
- API credentials;
- QuickBooks metadata;
- banking integration tokens;
- GitHub repository metadata;
- email integration settings.

### Operational Data
- logs;
- task execution metadata;
- automation status information;
- audit trails.

---

## 3. Local Processing

The Software is intended to run locally or within infrastructure controlled by the User.

Unless separately configured:
- Hush Line does not receive transaction-level accounting data;
- Hush Line does not continuously monitor bookkeeping records;
- Hush Line does not access banking credentials.

Users are responsible for:
- securing local systems;
- encrypting stored data;
- managing backups;
- protecting credentials.

---

## 4. Optional Network Services

If enabled by the User, the Software may communicate with:
- banking APIs;
- accounting providers;
- email providers;
- GitHub;
- cloud services;
- AI providers;
- notification systems.

Those services operate under their own privacy policies and terms.

Hush Line is not responsible for third-party data handling practices.

---

## 5. Telemetry and Diagnostics

If diagnostic or telemetry features are enabled, the Software may transmit limited operational information such as:
- software version;
- crash reports;
- configuration metadata;
- performance diagnostics.

Telemetry should not intentionally include transaction contents or sensitive financial records unless explicitly configured by the User.

---

## 6. AI Providers and External Models

If the User configures AI integrations:
- prompts and outputs may be transmitted to external AI providers;
- external providers may temporarily process submitted data;
- Users should avoid transmitting sensitive financial or personally identifiable information unless appropriate safeguards exist.

Users are responsible for evaluating the privacy and compliance posture of any AI provider they enable.

---

## 7. Data Retention

Because the Software primarily operates locally, retention periods are determined by the User.

Hush Line does not control retention of:
- local databases;
- accounting exports;
- backups;
- email archives;
- third-party integrations.

---

## 8. Security

Hush Line recommends:
- full-disk encryption;
- encrypted backups;
- least-privilege API credentials;
- hardware security modules where appropriate;
- MFA for financial systems;
- isolated execution environments;
- routine patching.

No software or infrastructure can be guaranteed completely secure.
